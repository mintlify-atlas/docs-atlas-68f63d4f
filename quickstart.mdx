---
title: "Get started: Add SlideMenu to a SwiftUI app"
description: "Add a gesture-driven side menu to a SwiftUI app with SlideMenu in under 5 minutes, from package installation through a fully styled, production-ready example."
---

This guide walks you through adding SlideMenu to a SwiftUI app, wiring up a toolbar toggle button, and applying appearance modifiers — ending with a complete, production-ready example you can drop straight into your project.

<Steps>
  <Step title="Install the package">
    Add SlideMenu via Swift Package Manager. In Xcode go to **File → Add Package Dependencies**, paste the URL below, and select the `main` branch.

    ```
    https://github.com/matteozappia/SlideMenu.git
    ```

    Then import the module in any file you want to use it:

    ```swift
    import SlideMenu
    ```

    For full installation instructions, see the [installation guide](/installation).
  </Step>

  <Step title="Wrap your content in SlideMenu">
    `SlideMenu` takes two view builder closures: `main` for your primary content and `menu` for the panel that slides in. Both are required.

    ```swift
    import SwiftUI
    import SlideMenu

    struct AppView: View {
        var body: some View {
            SlideMenu {
                Text("Main Content")
            } menu: {
                VStack {
                    Text("Home")
                    Text("Settings")
                }
            }
        }
    }
    ```

    By default the menu slides in from the left edge and responds to swipes within 30 points of the screen edge. Tapping the dimming overlay closes it.

    <Tip>
      Pass `direction: .right` to `SlideMenu` if you want the menu on the trailing edge of the screen.
    </Tip>
  </Step>

  <Step title="Add a SlideMenuButton to the toolbar">
    `SlideMenuButton` reads the toggle action from the SwiftUI environment automatically — you don't need a binding or a callback. Place it anywhere inside the `main` content closure, including inside a `NavigationStack` toolbar.

    ```swift
    SlideMenu {
        NavigationStack {
            Text("Hello")
                .toolbar {
                    ToolbarItem(placement: .topBarLeading) {
                        SlideMenuButton(systemImage: "sidebar.leading")
                    }
                }
        }
    } menu: {
        Text("Menu")
    }
    ```

    You can also provide a fully custom label using the view builder initializer:

    ```swift
    SlideMenuButton {
        Label("Menu", systemImage: "line.3.horizontal")
    }
    ```
  </Step>

  <Step title="Customize appearance with modifiers">
    All appearance settings are applied through view modifiers on the `SlideMenu` or any ancestor view. The modifiers write into the SwiftUI environment, so you can set defaults at the app root and override them locally.

    **Menu width**

    ```swift
    .slideMenuWidth(300)
    ```

    **Background** — accepts any `ShapeStyle`: solid colors, materials, or gradients:

    ```swift
    .slideMenuBackground(.ultraThinMaterial)
    .slideMenuBackground(Color(.systemBackground))
    .slideMenuBackground(LinearGradient(colors: [.blue, .purple], startPoint: .top, endPoint: .bottom))
    ```

    **Dimming overlay** — the overlay drawn over the main content while the menu is open:

    ```swift
    .slideMenuOverlayColor(.black)
    .slideMenuOverlayOpacity(0.4)
    ```

    **Border** — a thin stroke around the main panel while the menu is visible:

    ```swift
    .slideMenuBorderColor(.gray)
    .slideMenuBorderWidth(0.5)
    ```

    **Animation** — any SwiftUI `Animation`:

    ```swift
    .slideMenuAnimation(.spring(response: 0.35, dampingFraction: 0.85))
    .slideMenuAnimation(.bouncy(duration: 0.5, extraBounce: 0.15))
    .slideMenuAnimation(.easeInOut(duration: 0.3))
    ```

    <Note>
      Modifiers use default values when not applied. The defaults are: width `290`, background `.background`, overlay opacity `0.6`, border color `.primary`, border width `1`, and animation `.spring(response: 0.35, dampingFraction: 0.85)`.
    </Note>
  </Step>

  <Step title="Put it all together">
    Here is a complete example combining all the features above. It uses a left-side menu with a 40-point edge zone, a toolbar button, and a fully styled appearance:

    ```swift
    import SwiftUI
    import SlideMenu

    struct AppView: View {
        var body: some View {
            SlideMenu(direction: .left, edgeZoneWidth: 40) {
                NavigationStack {
                    ContentView()
                        .toolbar {
                            ToolbarItem(placement: .topBarLeading) {
                                SlideMenuButton(systemImage: "sidebar.leading")
                            }
                        }
                }
            } menu: {
                MenuView()
            }
            .slideMenuWidth(300)
            .slideMenuBackground(.ultraThinMaterial)
            .slideMenuOverlayOpacity(0.5)
            .slideMenuBorderWidth(0)
            .slideMenuAnimation(.bouncy(duration: 0.5, extraBounce: 0.15))
        }
    }
    ```

    `edgeZoneWidth: 40` gives users a slightly wider swipe target along the left edge. Setting `.slideMenuBorderWidth(0)` removes the border entirely for a cleaner look.

    <Tip>
      If you need menu items to navigate between views in the main content area, read `slideMenuToggle` from the environment in your menu view and call it after updating your selection state. This keeps the `NavigationStack` in the main panel where `NavigationLink` pushes work correctly.
    </Tip>
  </Step>
</Steps>
